Information For the User

The muon nearline processor find muons and missed muons by running the first pass of the two pass muon follower and the missed muon follower datacleaning processors. Muons are identified by their large NHit and high number of OWL hits. Missed muons are identified by looking for follower events after a 60NHit event. This table shows the number of each by run. For any run that is not a physics run rollover a fake muon is inserted at the beginning of the run due to the fact that we can't be certain that we didn't miss a muon right at the start of the run.
